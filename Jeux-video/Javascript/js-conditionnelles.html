Les conditionnelles

Dans le cas des scripts, des valeurs sont régulièrement affectées à des variables. Cela peut se faire par défaut, dès le chargement du script, soit par la saisie d'utilisateur, soit suite à une modification qui est intervenue à un moment donné. Le propre de la variable étant d'avoir une valeur changeante selon des critères, il sera important de pouvoir tester si la valeur actuelle d'une variable correspond à un besoin et faire poursuivre le script en fonction de cette valeur.
Il s'agit en quelque sorte d'orientée l'exécution du script dans certaines directions si une condition particulière est remplie. Quelques cas simples de ce principe :
  • un utilisateur saisi son nom et on vérifie qu'il fait partie d'une liste connue (qu'il est enregistré). Si c'est le cas, on lui donne accès à la page suivante, sinon, on lui propose de recommencer;
  • dans le cas d'une boutique, si la personne dépasse une certaine somme d'achat, on va lui offrir les frais de port; si c'est le jour de Noël, alors on affiche une image particulière sur la page d'accueil du site.
On pourrait lister comme ça un nombre infini de cas. Une grande partie du travail du développeur est de faire la liste des cas d'utilisations et des valeurs et de faire le programme qui correspondent à ces différentes situations. Les conditions sont certaines la chose la plus incontournable en programmation.

Si ... Alors ... Sinon ...

La solution la plus fréquent est d'utiliser if ... else ... pour faire le test. La condition à évaluer est placée entre parenthèses après le if et les actions à effectuer entre accolades sous la forme : 
if (condition){
   actions si la condition est vérifiée
}
else
{
   actions si la condition est invalidée
}
La partie sinon est optionnelle et il est possible d'écrire : 
if (condition){
   actions si la condition est vérifiée
}
La condition sera explicitée sous la forme d'une comparaison (<, <=, ==, !=, >=, >) ou d'un appel de fonction. 
var a, b;
...
if ( a < b ) {
    document.write ( "a est le plus petit" );
} else if ( a > b ) {
    document.write ( "b est le plus petit" );
} else {
    document.write ( "a et b sont égaux" );
}

selon ... cas ...

Les instructions switch servent en général à tester si une valeur est vérifier, à agir en conséquence dans ce cas et différemment dans un autre cas. Mais il arrive régulièrement que le nombre de cas augmente t qu'ils soient identifiables. Par exemple, dans le cas d'une boutique en ligne, les envois peuvent être effectués dans divers pays et les frais ports pourront varier. Il est donc important de pouvoir tester la zone géographique vers laquelle le transport doit être fait pour appliquer le bon tarif.
Dans ces la condition switch ... case est souvent plus pratique. Elle permet de définir après le switch la valeur à tester. Ensuite chaque valeur possible sera listée et précédée du mot-clé case. Une instruction cas doit se terminer par un : pour débuter le bloc d'instruction spécifique à ce cas. 
var pays = "France";
switch (pays){
case "France":
    document.write("envoi pour la France à 5euros");
    break;
case "Allemagne":
    document.write("envoi pour l'Allemagne à 7euros");
    break;
default :
   document.write("envoi pour les autres pays  à 10euros");
    break;
}
Attention, si on enlève le break et que la condition est remplie, javascript exécute l'ensemble des autres cas qui suivent. Break sert donc à sortir de la liste des cas après les premier cas validé. On pourrait en effet imaginer qu'une situation corresponde à plusieurs cas. Break évite alors les surprises.

Associer plusieurs tests et conditions en une

Il est possible d'un cas corresponde à plusieurs situations. Dans ce cas, les situations peuvent être une intersection des cas (cas se recouvrant ou les deux devant être vraies) ou un choix (dont l'une des deux sera vraie) :
  • l'opérateur && (et) permet d'associer deux conditions impliquant que la valeur a tester doit corresponde aux deux
  • l'opérateur || (ou) permet de vérifier que la valeur correspond à l'une ou l'autre des situations
par exemple si la Belgique a les mêmes frais e port que la France, nous pourrions écrire :
